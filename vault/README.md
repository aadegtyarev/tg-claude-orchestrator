# vault — автономный кошелёк секретов

Держит секреты оператора вне досягаемости модели: значение секрета живёт на хосте,
а модель получает лишь авторизованный результат. Пакет без зависимостей
оркестратора (нет `aiogram`/Telegram) — работает и без бота. Оркестратор является
клиентом через тонкий модуль-адаптер. Фаза 1 редизайна, см.
`docs/ARCHITECTURE-claude-box.md` §4 и `docs/DECISIONS-claude-box.md`.

Пользовательский гайд (настройка secrets.toml, режимы, примеры) —
`docs/secrets-wallet.md`. Этот README — про устройство пакета.

## Как модель получает доступ, не видя секрета (тиры)

- **Тир 1 — шимы**: команду (`git push`, `gh`, `curl`) перехватывает PATH-обёртка
  и исполняет на хосте с реальным кредом; значение в песочницу не входит. Клиент
  внутри песочницы — stdlib-only `bin/wallet` (venv там недоступен).
- **Тир 2 — MITM-прокси**: HTTPS к сервису идёт через локальный прокси кошелька,
  который подставляет кред на проводе по коннектору. Значение не входит; модель
  ходит обычным запросом.
- **Тир 3 — минт** (escape hatch): где сервис умеет суб-токены (Jenkins, GitLab) —
  выпускается короткоживущий скоупнутый под-токен; единственный тир, где значение
  у модели, поэтому и только минт, не мастер-секрет.

## Коннекторы

Плагин, знающий конкретный сервис: `authorize` (подставить кред) и `in_scope`
(ALLOW | DENY | ASK по запросу). Контракт — `connectors/contract.py`, реестр —
`connectors/__init__.py`. Есть: `generic-bearer` (bearer-токен + скоуп по
URL-префиксам), `gdocs` (Google Docs, deny share/export вне скоупа; live-OAuth —
следующий трек). ASK поднимает спрос гранта у оператора (§4.6); DENY обязан нести
предписывающий `remedy` (Р0). Новый коннектор = модуль + `register()`.

## Демон и решение доступа

`daemon.py` — HTTP-API секретов (`/run`, `/exec`, `/get`), `proxy.py` — MITM-прокси
тира 2. Решение policy до подтверждения — `verdict.py` (allowed/needs_confirm),
скоуп-решение запроса — коннектор (`ScopeVerdict`). Скоуп прокси читается из
`secrets.toml` живьём (по mtime): грант и отзыв действуют без перезапуска;
`rm secrets.toml` отзывает доступ, ошибка чтения держит последний валидный scope
(fail-safe, не расширяет).

## CLI `vault`

```
vault serve   # поднять демон standalone (без оркестратора), записать ~/.wallet.json
vault policy  # просмотр/правка secrets.toml (new/cmd/sessions/deny/confirm/scope/rm)
```

`serve` использует `tty_host.py` (вопросы confirm/ASK в терминал). Клиентские
команды (`ls/run/exec/get/env`) — в stdlib-only `bin/wallet`, он работает и внутри
песочницы. Запуск — `bin/vault` или `python -m vault`.

## Запись policy — безопасность

`policy.py` пишет `secrets.toml` через `mkstemp`+`os.replace` (симлинк-безопасно,
права 0600) под межпроцессным `flock`; значения валидируются (полный URL, без
глоба/управляющих байт). Значения токенов через бота/CLI-policy не трогаются —
только правкой host-файла (иначе токен утёк бы в историю).

## Автономность и тесты

Пакет не тянет оркестратор; `box_cli`/CLI используют только `vault.*`. Тесты —
`tests/vault_*_test.py`, `tests/wallet_*_test.py`, `tests/ask_grant_test.py`,
`tests/proxy_scope_reload_test.py`. Прогон: `.venv/bin/python -m pytest -q` и
`PY=.venv/bin/python tests/run_all.sh`.
