# Changelog

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версии — [SemVer](https://semver.org/lang/ru/). До 1.0 публичный контракт
(команды бота, формат `secrets.toml`, `.env`) может меняться в minor-релизах.

## [0.2.0] — 2026-07-20

### Добавлено
- **Кошелёк секретов: прозрачный шлюз.** Модель зовёт `gh`/`git`/`curl`/`ssh`
  как обычно — per-session обёртки в `.wallet-bin` (первыми в PATH песочницы)
  заворачивают вызов в `wallet exec`, демон подбирает секрет по команде и
  исполняет её на хосте. Больше не нужно вручную помнить про `wallet run`.
  - `git`-обёртка особая: сетевые подкоманды (`push`/`fetch`/`pull`/`clone`/…)
    идут через кошелёк, локальные (`status`/`commit`/`diff`) — настоящим git в
    песочнице.
  - inject-токен виден модели привычной env-переменной `$NAME` (в песочнице —
    маркер, реальное значение демон подставляет на хосте; `$NAME_FILE` — для
    ключа/сертификата файлом).
  - `SessionManager.path_hooks` — модули добавляют каталоги в начало PATH
    песочницы.
  - CLI: новая команда `wallet exec <cmd>` (подбор секрета по команде).

### Изменено
- Системный промпт и справка кошелька переписаны: ведут с «зови инструмент как
  обычно», `wallet run` — ручной fallback.
- Документация `docs/secrets-wallet.md` переписана с нуля под прозрачную модель;
  README и `.env.example` актуализированы.

### Удалено
- Плейсхолдеры `{{secret}}`/`{{secret_file}}` — заменены на env-переменные
  `$NAME`/`$NAME_FILE`.
- Поле секрета `inject_at_start` (shared-секрет с `env` теперь всегда попадает в
  env сессии реальным значением).

## [0.1.0]

Базовая линия до введения версионирования: ядро оркестратора (сессии, бабблы,
ход хода), адаптеры Telegram и Web, раннеры (bwrap/agent-vm/off), файловая
песочница bubblewrap, модуль кошелька секретов (host/inject/shared, policy,
`/wallet`), UX статус-бабла.
