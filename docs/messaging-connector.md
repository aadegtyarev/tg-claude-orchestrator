# Транспорт-коннектор: статус — реализован (2026-07)

Изначальный набросок этого документа («выделить из bot.py интерфейс
Transport») выполнен: ядро и транспорты разделены. Документ оставлен как
ориентир для СЛЕДУЮЩИХ адаптеров (Matrix и др.).

## Что сделано

- `orchestrator/core/transport.py` — протокол `Transport`: доставка текстов
  и файлов, статус-бабл (post/edit/finish/freeze), typing, permission-кнопки,
  привязка сессий (`bind_session`/`unbind_session`).
- `orchestrator/core/app.py` — `OrchestratorCore`: командный слой и
  маршрутизация общие; ядро шлёт события во ВСЕ активные адаптеры разом.
- `orchestrator/adapters/telegram/` — Telegram стал первым адаптером;
  `orchestrator/adapters/web/` — второй (локальный веб-интерфейс).
- Выбор — `ADAPTERS=telegram,web` в `.env`; одна сессия видна из всех
  адаптеров параллельно (bindings per-адаптер в записи сессии).
- `context_id = <адаптер>:<имя-сессии>:<токен>` — происхождение сообщения;
  reply-цитата достаётся адаптеру-источнику, остальные получают ответ без неё.

## Как добавить новый адаптер (например, Matrix)

1. Подпакет `orchestrator/adapters/matrix/` с классом, реализующим
   `Transport` (образец — telegram/adapter.py: команды пользователя адаптер
   переводит в вызовы ядра, доставку реализует своими средствами).
2. Ветка в `adapters/make_adapters()` + имя в `config._parse_adapters`.
3. Всё. Ядро, сессии, бабл, permission relay и тексты уже общие.

## Памятка по соответствию понятий (для Matrix)

| Ядро | Telegram | Matrix |
|------|----------|--------|
| Сессия (bind_session → адрес) | форум-топик | комната или thread |
| Статус-бабл (ref) | редактируемое сообщение | событие + `m.replace` |
| Кнопки (permission, стоп) | inline-кнопки | реакции/reply-команды (нативных нет) |
| «печатает…» | chat action | typing notification |
| Белый список | user id | MXID |

Неудобство Matrix — отсутствие inline-кнопок: permission relay и «📋/⛔»
придётся эмулировать реакциями (✅/❌) или текстовыми ответами; протокол ядра
этого не запрещает — `permission_prompt` волен рисовать что угодно, вердикт
возвращается через `core.permission_verdict`.
